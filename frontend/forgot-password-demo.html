How It Works

This demo shows the complete forgot password flow:

  1. Request Reset: User enters email on forgot password page
  2. Generate Token: Backend creates a secure reset token
  3. Reset Password: User uses token to set new password

Test the Flow

1. Forgot Password Page 2. Reset Password Page 3. Login Page 4. Register Page

Testing Instructions

Step 1: Go to "Forgot Password Page" and enter an email

Step 2: Check the browser console for the reset token

Step 3: Copy the token and go to "Reset Password Page"

Step 4: Add ?token=YOUR_TOKEN_HERE to the URL

Step 5: Enter new password and test the reset

Example Reset URL

After getting a token from the forgot password page, your reset URL should look like:

http://localhost:5000/reset-password?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Backend API Endpoints

POST /api/auth/forgot-password - Request password reset

POST /api/auth/reset-password - Reset password with token

POST /api/auth/login - Login with new password

Security Features

  • Reset tokens expire after 1 hour
  • Tokens are single-use only
  • Passwords are hashed with bcrypt
  • No email/user existence disclosure
